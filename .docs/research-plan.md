# Multica 开源项目深度研究计划

## 一、项目概述

Multica 是一个开源的 AI-native 任务管理平台，被称为"像 Linear 一样的工具，但 AI agents 是第一等公民"。该项目的主要特点是将 AI 编程 agent 转化为真正的团队成员，可以分配任务、跟踪进度、累积技能。

### 1.1 项目基本信息

- **项目名称**: Multica
- **项目类型**: 开源 SaaS 平台 (AI Agent 管理平台)
- **官方网站**: https://multica.ai
- **GitHub**: https://github.com/multica-ai/multica
- **许可证**: MIT License
- **核心功能**: AI Agent 任务管理、技能复用、多工作区支持

### 1.2 支持的 AI Agent

Multica 支持多种 AI Agent CLI 的集成:

- Claude Code
- Codex
- OpenClaw
- OpenCode
- Hermes
- Gemini
- Pi
- Cursor Agent

## 二、研究目标

本研究的总体目标是全面理解 Multica 项目的技术实现，包括:

1. **整体架构**: 理解前后端技术栈、数据流、模块划分
2. **核心功能实现**: 深入研究关键业务功能的实现细节
3. **代码质量分析**: 代码风格、测试覆盖、技术债务
4. **前后端交互**: API 设计、WebSocket 通信、状态管理

## 四、研究方法

### 4.1 静态代码分析

- 阅读核心源代码文件
- 分析模块间的依赖关系
- 理解关键数据结构和算法

### 4.2 动态运行分析

- 运行 `make dev` 启动开发环境
- 分析请求流程和数据流转
- 测试关键功能点

### 4.3 文档研究

- 阅读 README、CLAUDE.md 等项目文档
- 分析 API 文档和数据库 schema
- 查看测试用例
